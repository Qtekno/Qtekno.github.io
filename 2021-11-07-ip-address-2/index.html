IP Address 2

Publish date: 2021-11-07
Tags: IPV4

Dalam dunia Jaringan Komputer dikenal 2 jenis model networking, yaitu Model Referensi OSI dan Model TCP/IP.

Model OSI mendefinisikan framework networking dalam bentuk 7 layer. Kontrol data diserahkan dari satu layer ke layer berikutnya, mulai dari layer aplikasi turun sampai layer terendah (layer fisik). Kontrol data kemudian dikirimkan melintasi koneksi fisik antara setiap titik sepanjang jalur perjalanan kemudian diserahkan kepada ke layer paling atas pada komputer penerima. OSI sendiri merupakan singkatan dari Open System Interconnection. Model ini disebut juga dengan model Model tujuh lapis OSI" (OSI seven layer model)

Model TCP/IP (Transmission Control Protocol and Internet Protocol) adalah standar komunikasi data yang digunakan oleh komunitas internet dalam proses tukar-menukar data dari satu komputer ke komputer lain di dalam jaringan Internet. Protokol TCP/IP dikembangkan pada akhir dekade 1970-an hingga awal 1980-an sebagai sebuah protokol standar untuk menghubungkan komputer-komputer dan jaringan untuk membentuk sebuah jaringan yang luas (WAN).

TCP/IP merupakan sebuah standar jaringan terbuka yang bersifat independen terhadap mekanisme transport jaringan fisik yang digunakan, sehingga dapat digunakan di mana saja. Protokol TCP/IP menggunakan skema pengalamatan yang sederhana yang disebut sebagai alamat IP (IP Address) yang mengizinkan komputer untuk dapat saling berhubungan satu sama lainnya di Internet. IP juga bersifat routable yang berarti protokol ini cocok untuk menghubungkan sistem-sistem berbeda (seperti Microsoft Windows dan keluarga UNIX) untuk membentuk jaringan yang heterogen.

Sistem protokol TCP/IP dibagi-bagi menjadi 4 (atau 5) layer yang masing-masing menjalankan tugas yang spesifik. Model berupa protocol stack (tumpukan protokol layer) ini bermula sejak pertama kali TCP/IP dibuat. Berikut adalah layer-layer protokol TCP/IP secara official. Sistem 4 Layer tersebut adalah:

  1. Network Interface Layer (Physical Layer dan Datalink Layer)
  2. Network Layer
  3. Transport Layer
  4. Application Layer

Dalam jaringan model TCP/IP dikenal 3 alamat yakni: physical address, Logical Address (IP address) dan port address. Berikut penjelasannya:

  1. Pysical Address (tergantung NIC) - Pengalamatan Layer 2 OSI

Alamat fisik adalah system pengalamatan pada jaringan komputer yang dikenal dengan nama Media Access Control Address atau MAC address. Disebut dengan MAC Address karena alamat ini di hard-code ke dalam network interface controller (NIC) dari perangkat layer fisik yang terhubung pada network.

Layer Data Link menggunakan address fisik atau hardware untuk memastikan data diterima pada mesin yang tepat dalam LAN. Setiap MAC address harus unik dan menggunakan format sebagai berikut :

  • Terdiri dari 48 bit (6 byte). Penomoran MAC Address Dituliskan dalam bentuk 12 digit hexadecimal (0-9, A-F). Dengan penomoran ini akan hanya membutuhkan 12 field data.
  • 6 digit hexadecimal pertama dalam address menunjukkan kode vendor atau unique identifier (OUI) dan 6 digit hexadecimal terakhir diberikan oleh pabrik NIC dan harus unik untuk semua nomor yang diberikan oleh pabrik tersebut. Contoh sebuah MAC address dapat berupa 00:00:07:A9:B2:EB. OUI-nya adalah 00:00:07.
  • Setiap perangkat komputer dan perangkat jaringan mulai dari router, wifi, smartphone, laptop, server yang terhubung dengan jaringan yang terpasang NIC akan mempunyai MAC address yang berbeda-beda satu sama lain.
  • MAC Address adalah sistem pengalamatan yang bersifat permanen dan tertanam pada network interface card dan digunakan pada Layer 2.
  1. IP Address

Internet Protocol Address (atau disingkat alamat IP) adalah label numerik yang ditetapkan untuk setiap perangkat yang terhubung ke jaringan komputer yang menggunakan Protokol Internet untuk komunikasi. Alamat IP memiliki dua fungsi utama: host atau identifikasi antarmuka jaringan dan pengalamatan lokasi.

IP Address merupakan pengalamatan pada Layer 3 OSI (Network Layer) dan dikenal sebagai alamat logika. Administrator jaringan menetapkan alamat IP untuk setiap perangkat yang terhubung ke jaringan. Penugasan semacam itu mungkin bersifat statis (tetap atau permanen) atau dinamis, tergantung pada praktik jaringan dan fitur perangkat lunak.

  1. Port Address(16 bit)

Port Address dibutuhkan untuk dapat menjalankan banyak aplikasi/ proses pada saat yang bersamaan. Alamat port digunakan pada Layer 4 OSI (Layer Transport). Alamat nomor port dibatasi dalam 16 bit. Nomor port yang benar berada dalam range 0 s.d 65.535 (2^16 = 65.536). Sehingga secara teori jumlah client yang dapat dihubungkan dengan sebuah server yang sama pada suatu saat tertentu adalah sebanyak 65.536.
